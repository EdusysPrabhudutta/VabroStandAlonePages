Get Started

Here's how you can use
Vabro for Scrum Projects

Create Project

Product Backlog

You can create a backlog of Epics (Optional), User Stories, Tasks in Vabro and prioritize them for future sprints. Use a task management form or template for the product backlog, adding user stories, feature descriptions, and other backlog items.

Sprint Planning

For each sprint, you can select items from the backlog and assign them to the current sprint. Create a sprint-specific project or board where team members can focus on the current sprint's tasks.

Manage Sprints

Sprint Planning and Task Assignment

In Vabro, create tasks for each user story or feature you want to complete in the sprint. Assign these tasks to team members based on their skills and availability.

You can use task templates in Vabro to define recurring Sprint tasks such as Sprint Backlog Creation, Sprint Review, Sprint Retrospective, etc.

Track Sprint Progress

Use Kanban boards or task lists to monitor the progress of sprint tasks in real time. Track task completion and update task statuses (e.g. To Do, In Progress, Completed).

Burndown Charts

While Vabro might not have built-in Scrum-specific tools like burndown charts, you can manually track progress by using task completion percentages or integrate with other tools to get burndown charts if necessary.

Daily Standups

Task Comments

During daily standups, use Vabro's task comments to provide updates and mark obstacles or dependencies. Each team member can update their assigned tasks with the current progress and blockers.

Automated Reminders

Set up reminders or notifications for the team to check on the progress of tasks or for the Scrum Master to keep track of task updates.

Sprint Review

Review Completed Tasks

At the end of each sprint, review all the completed tasks within Vabro. You can track whether the work is aligned with the sprint goals and identify any remaining items.

Feedback Forms

Use feedback forms to collect insights from stakeholders during the sprint review. This can help your team refine and adjust the backlog items or sprint goals for future sprints.

Sprint Retrospective

Collect Retrospective Data

Use Vabro's forms or surveys to collect feedback from team members on what went well, what didn't, and what could be improved. These forms can be structured to capture specific retrospective questions (e.g. What went well?, What can we improve?, What will we commit to for the next sprint?).

Action Items

Based on the retrospective feedback, create tasks or action items for the next sprint, ensuring continuous improvement and refinement of the Scrum process.

Managing Backlog & Prioritization

Product Backlog

Use Vabro to prioritize the product backlog based on business value, team capacity, and deadlines. Create a backlog management workflow that automatically reassigns tasks, reorders them, or flags items for future sprints.

User Stories and Epics

If Vabro supports hierarchical task management, you can structure your work using user stories and epics to align with the Scrum approach, helping the team manage large projects in bite-sized, manageable parts.

Automating & Streamlining Scrum Processes

Workflow Automation

Vabro allows you to automate certain tasks, such as sending reminders for upcoming sprint reviews or updating task statuses once the previous tasks are marked complete. This can be valuable for Scrum projects where consistency and time management are key.

Forms for Task Submissions

Use custom forms for story creation, task updates, and bug reporting. This ensures that the team follows consistent procedures for creating and updating tasks.

Reporting and Metrics

Progress Tracking

Use Vabro's reporting and tracking features to monitor the team's performance during sprints. You can track how many tasks are completed and how many are in progress, providing insights into sprint health.

Task Completion Tracking

You can manually calculate or automate task completion rates within Vabro, which will help with Scrum metrics like velocity (i.e., the number of story points completed in each sprint).

Retrospective Reports

After each sprint, you can create retrospective reports based on feedback and task outcomes. This will help your Scrum team analyze performance and identify areas for improvement.
